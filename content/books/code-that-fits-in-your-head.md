A practical guide to writing sustainable software using heuristics that help manage complexity. Mark Seemann draws on decades of experience to show how to structure code so it remains understandable as projects grow, using C# examples throughout.

The book covers topics like checklists for new codebases, encapsulation, decomposition, and API design, all tied together by the idea that code should be simple enough to reason about without external aids.
